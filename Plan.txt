# TextToSpeech Project Plan

Current implementation uses a local Python pipeline built around the Kokoro texttospeech model. The project no longer relies on external TTS providers; everything runs offline once the Kokoro models are downloaded.

---

## System Overview
The agent operates in four sequential stages:

1. **Text analysis**  parse raw input text into segments with speaker metadata using an LLM (Gemini) or a fallback rulebased parser.
2. **Voice assignment**  map each segment to a voice identifier exposed by the TTS package and optionally adjust speed.
3. **Audio generation**  synthesize each segment via the selected TTS library, returning numpy arrays at 24kHz.
4. **Stitch & export**  concatenate the clips with configurable silence and write a 16bit PCM WAV using Pythons builtin wave module.

Network access is only required for the optional Gemini API key; all TTS is local with the chosen TTS library.

## Stage3  Kokoro Synthesis

The TTS backend is initialized lazily via its package API (e.g. a preload or constructor call). _generate_segment simply calls

`python
from <tts_package> import generate_audio, SAMPLE_RATE  # replace with real import

...

def _generate_segment(text, voice, speed):
    return generate_audio(text, voice=voice)
`

which returns a float32 numpy array at 24000Hz. Errors are caught to allow the pipeline to continue if a segment fails.

---

## Stage4  Stitching & Export

This logic is unchanged from the previous design: silence segments are inserted and the final audio concatenated, clamped, and written out using wave. Output metadata is printed.
