System Overview
The agent has three core stages: Analyze → Script → Synthesize
Stage 1: Text Analysis (LLM)
Feed the input text to Gemini-2.5-flash with a prompt that extracts:

Characters and speakers — names, gender, age clues, personality traits
Narrative tone — formal novel, casual article, dramatic story, etc.
Dialogue vs narration — who says what, and what's just narration
Emotional cues — anger, sadness, excitement (for TTS emotion tags)

Output is a structured script with speaker metadata and emotion labels.
Stage 2: Script Structuring
Transform the labeled text into a voice script with speaker, gender, age, emotional style, and text.
Stage 3: Voice Synthesis (TTS)
Use Bark TTS for each segment. Bark supports emotion tags like [laughs], [sighs], [whispers], [shouts], [crying] which Gemini can generate automatically. Runs fully local after initial model download — no network calls during synthesis, no hanging issues.
Stage 4: Audio Stitching
Use pure Python byte concatenation to stitch MP3 segments. No pydub, no ffmpeg — completely eliminates hanging problems.
Full Tech Stack
Gemini → Script analysis and emotion tagging
Bark TTS → Local emotional voice synthesis
Python script → Pure-Python MP3 concatenation
Streamlit app → User interface
