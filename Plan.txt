System Overview
The agent has three core stages: Analyze → Script → Synthesize
Stage 1: Text Analysis (LLM)
Feed the input text to an LLM (like Gemini-2.5-flash) with a prompt that extracts:

Characters/speakers — names, gender, age clues, personality traits
Narrative tone — formal novel, casual article, dramatic story, etc.
Dialogue vs narration — who says what, and what's just narration

Example prompt:

"Analyze this text. Identify every speaker, their likely gender/age/tone, and label each segment as [NARRATOR], [CHARACTER: John - gruff older man], etc."

The output is a structured script — a list of segments with speaker metadata.

Stage 2: Script Structuring
Transform the labeled text into a voice script like:
json[
  { "speaker": "narrator", "gender": "neutral", "style": "calm, storytelling", "text": "It was a dark night..." },
  { "speaker": "John", "gender": "male", "age": "old", "style": "gruff, tired", "text": "I can't do this anymore." },
  { "speaker": "Sarah", "gender": "female", "style": "anxious, fast-paced", "text": "You have to, John!" }
]
```

### Full Tech Stack Recommendation
```
Input Text
    ↓
[Gemini] — Analyze & label speakers + generate structured script
    ↓
[Python script] — Assign voice IDs to each character (male/female/old/young)
    ↓
[LLM TTS] — Generate audio per segment with matching voice
    ↓
[pydub / ffmpeg] — Stitch all audio segments into one final audio file
    ↓
Output: podcast-style MP3/WAV

Key Implementation Steps
1. Speaker-to-voice mapping — Create a mapping like:

Male narrator → ElevenLabs "Adam" voice
Old gruff man → ElevenLabs "Arnold" voice
Young woman → ElevenLabs "Rachel" voice

2. SSML / style prompting — ElevenLabs lets you pass "voice settings" (stability, expressiveness) per segment. You can also prefix the text with style hints like "[tired, whispering]" before generating.
3. Stitching — Use pydub in Python to concatenate audio clips with small pauses between speaker changes.

Minimal Python Pseudocode
pythonimport anthropic, requests, pydub
